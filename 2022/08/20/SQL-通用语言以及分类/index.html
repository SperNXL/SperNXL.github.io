SQL-通用语言以及分类

Author: Arno Dorian Date: 八月 20, 2022  0:24:49 Category: MySQL

SQL通用语法

  • SQL语句可以单行或多行，以分号结尾
  • SQL语句可以使用空格/缩进来增强语句可读性
  • MySQL数据库的SQL语句部分大小写，关键字建议大写
  • 注释：
    • 单行注释：–注释内容或#注释内容（MySQL特有）
    • 多行注释 /* 注释内容 */

SQL分类

  • DDL (Data Definition Language) 数据定义语言
  • DML(Data Manipulation Language) 数据操作语言
  • DQL(Data Query Language) 数据查询语言
  • DCL(Data Control Language) 数据控制语言

    类型 大小 有符号范围 无符号范围 描述
    TINYINT 1byte (-128,127) （0，255） 小整数值
    SMALLINT 2byte (-32768,32767) （0，65535） 大数数值
    MEDIUMINT 3byte (-8388608,8388607) （0，16777215） 大数数值
    INT 或INTEGER 4byte (-2147483638,2147483647) （0，4294967295） 大数数值
    BIGINT 8byte (-2^63,2^63-1) （0，2^64-1） 极大数数值
    FLOAT 4byte (-3.402823466 E+38,3.402823466351 E+38) 0和(1.175494351 E-38,3.402823466 E+38) 单精度浮点数值
    DOUBLES 8byte -1.7976931348623157 E+308,1.7976931348623157 E+308 0和2.2250738585072014 E-308,1.7976931348623157 E+308 双精度浮点数值
    DECIMAL 依赖于M（精度）和D（标度）的值 依赖于M（精度）和D（标度）的值 小数值（精确定点数）

    • 字符串类型

      类型 大小 描述
      char 0-255byte 定长字符串
      varchar 0-65535byte 变长字符串
      tinyBlob 0-255byte 不超过255个字符的二进制数据
      tinyText 0-255byte 短文本字符串

      text 0-65 535byte 长文本数据
      mediumBlob 0-16 777 215byte 二进制形式的中等长度文本数据
      mediumText 0-16 777 215byte 中等长度文本数据
      longBlob 0-4 294 967 295byte 二进制形式的极大数据文本
      longText 0-4 294 967 295byte 极大文本数据

    • 日期类型

      类型 大小 格式 描述
      data 3 YYYY-MM-DD 日期体
      time 3 HH:MM:SS 时间值或持续时间
      year 1 YYYY 年份值
      dataTime 6 YYYY-MM-DD HH:MM:SS 混合日期和时间
      timeStamp 4 YYYY-MM-DD HH:MM:SS 混合日期和时间值，时间戳

DML

功能：用来对数据库中表的数据记录进行增删改操作。

  • 添加数据： insert
  • 修改数据:：update
  • 删除数据： delete
DML-添加数据

1.给指定字段添加数据

2.给全部字段添加数据

3.批量添加数据

注意：

  • ​ 插入数据时，指定的字段顺序需要与值的顺序是一 一对应的。
  • ​ 字符串和日期型数据应该包含在引号中。
  • ​ 插入的数据大小，应该在字段的规定范围内。
DML-修改数据

注意：修改语句的where条件可以有，也可以没有，如果没有条件，则会修改整张表的所有数据。

DML-删除数据

注意：

  • delete语句的where条件可以有，也可以没有，如果没有条件，则会删除整张表的所有数据。
  • delete语句不能删除某一个字段的值（可以使用update）。

2.where 和 having 区别

  • 执行时机不同：where 是分组之前进行过滤，不满足 where 条件，不参与分组；而having是分组之后对结果进行过滤。

  2. 创建用户

DCL-权限控制

MySQL中定义了很多种权限，但是常用的就一下这几种：

权限 说明
all,all privileges 所有权限
select 查询数据
insert 插入数据
update 修改数据
delete 删除数据
alter 修改表
drop 删除数据库/表/视图
create 创建数据库/表

  1. 查询权限

  2. 授予权限

  3. 撤销权限

注意：

  • 多个权限之间，使用逗号分隔。
  • 授权时，数据库名和表名可以使用*进行通配，代表所有。
